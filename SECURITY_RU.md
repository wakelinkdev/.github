# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Сообщение об уязвимости

**Пожалуйста, не сообщайте об уязвимостях безопасности через публичные GitHub issues.**

Вместо этого отправьте сообщение по электронной почте на **security@wakelinkdev.org**.

Вы должны получить ответ в течение 48 часов. Если по какой-либо причине этого не произошло, пожалуйста, отправьте повторное письмо, чтобы убедиться, что мы получили ваше первоначальное сообщение.

Пожалуйста, включите в свой отчёт следующую информацию:

- Тип проблемы (например, переполнение буфера, SQL-инъекция, межсайтовый скриптинг и т. д.)
- Полные пути к исходным файлам, связанным с проявлением проблемы
- Расположение затронутого исходного кода (тег/ветка/коммит или прямой URL)
- Специальная конфигурация, необходимая для воспроизведения проблемы
- Пошаговые инструкции по воспроизведению проблемы
- Proof-of-concept или код эксплойта (если возможно)
- Влияние проблемы, включая описание того, как злоумышленник может ею воспользоваться

## Политика раскрытия информации

- Мы ответим на ваш отчёт в течение 48 часов с нашей оценкой и ожидаемой датой устранения
- Мы будем держать вас в курсе прогресса в устранении уязвимости
- Мы уведомим вас, когда уязвимость будет устранена
- Просим вас дать нам 90 дней на решение проблемы до любого публичного раскрытия

## Безопасная гавань

Мы поддерживаем принцип безопасной гавани для исследователей безопасности, которые:

- Добросовестно стараются избежать нарушений конфиденциальности, уничтожения данных, прерывания или деградации наших сервисов
- Взаимодействуют только с аккаунтами, которыми владеют, или с явного разрешения владельца аккаунта
- Не используют уязвимость безопасности в целях, отличных от верификации
- Незамедлительно сообщают о любой обнаруженной уязвимости

## Признание заслуг

Мы верим в признание заслуг исследователей безопасности, которые помогают поддерживать безопасность WakeLink:

- Упоминание в наших рекомендациях по безопасности (по желанию)
- Внесение в наш Зал славы (скоро)
- Возможное денежное вознаграждение за критические уязвимости (рассматривается индивидуально)

## Обновления безопасности

Обновления безопасности будут выпускаться как патч-версии и объявляться через:

- GitHub Security Advisories
- Примечания к выпуску
- Электронная почта зарегистрированным пользователям (для критических проблем)
