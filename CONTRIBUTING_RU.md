# Участие в разработке WakeLink

Прежде всего, спасибо за желание внести вклад в WakeLink! Именно такие люди, как вы, делают WakeLink отличным инструментом.

## Кодекс поведения

Этот проект и все его участники руководствуются нашим Кодексом поведения. Участвуя в проекте, вы обязуетесь его соблюдать.

## Как я могу помочь?

### Сообщение об ошибках

Прежде чем создавать отчёт об ошибке, проверьте существующие issues — возможно, вам не нужно создавать новый. При создании отчёта об ошибке укажите как можно больше деталей.

**Сообщайте об ошибках в соответствующем репозитории:**

| Тип проблемы | Репозиторий |
|:-----------|:-----------|
| Криптография / Протокол | [ewsp-core](https://github.com/wakelinkdev/ewsp-core/issues) |
| Прошивка ESP | [firmware](https://github.com/wakelinkdev/firmware/issues) |
| CLI клиент | [cli](https://github.com/wakelinkdev/cli/issues) |
| Приложение Android | [android](https://github.com/wakelinkdev/android/issues) |
| Мультиплатформенное приложение | [multiplatform](https://github.com/wakelinkdev/multiplatform/issues) |
| Документация | [docs](https://github.com/wakelinkdev/docs/issues) |

### Предложения по улучшению

Предложения по улучшению отслеживаются как GitHub issues. Создайте issue в соответствующем репозитории и укажите:

- Чёткое и понятное название
- Подробное описание предлагаемого улучшения
- Объяснение, почему это улучшение будет полезным

### Pull Request'ы

1. Сделайте fork репозитория и создайте ветку от `main`
2. Если вы добавили код, который нужно протестировать, добавьте тесты
3. Если вы изменили API, обновите документацию
4. Убедитесь, что набор тестов проходит
5. Убедитесь, что ваш код проходит линтинг
6. Отправьте pull request!

## Настройка среды разработки

### ewsp-core (C)

```bash
git clone https://github.com/wakelinkdev/ewsp-core.git
cd ewsp-core
mkdir build && cd build
cmake .. -DEWSP_BUILD_TESTS=ON
cmake --build .
ctest -V
```

### wakelink-client (Python)

```bash
git clone https://github.com/wakelinkdev/cli.git
cd cli
pip install -e ".[dev]"
pytest
mypy core/
```

### wakelink-android (Kotlin)

```bash
git clone https://github.com/wakelinkdev/android.git
cd android
./gradlew build
./gradlew test
```

## Стиль кода

### Сообщения коммитов Git

- Используйте настоящее время («Add feature», а не «Added feature»)
- Используйте повелительное наклонение («Move cursor to...», а не «Moves cursor to...»)
- Ограничьте первую строку 72 символами или меньше
- Ссылайтесь на issues и pull request'ы после первой строки

**Типы коммитов:**
- `feat:` Новая функция
- `fix:` Исправление ошибки
- `docs:` Изменения только в документации
- `style:` Изменения стиля кода (форматирование и т. д.)
- `refactor:` Изменение кода, которое не исправляет ошибку и не добавляет функцию
- `perf:` Улучшение производительности
- `test:` Добавление недостающих тестов
- `chore:` Изменения в процессе сборки или вспомогательных инструментах

### Стиль кода C (ewsp-core)

- 4 пробела для отступов
- Имена функций: `ewsp_snake_case()`
- Макросы: `EWSP_UPPER_CASE`
- Все публичные функции должны иметь документацию

### Стиль кода Python (wakelink-client)

- Следуйте PEP 8
- Используйте аннотации типов
- Docstring для всех публичных функций
- Запускайте `black` и `isort` перед коммитом

### Стиль кода Kotlin

- Следуйте соглашениям по написанию кода Kotlin
- Используйте `ktlint` для форматирования
